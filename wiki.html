Experience points

  • Contributing to open source projects especially on github.
  • Insight into CI.
  • Insight into software-testing
  • Inisght into coding standards
  • Code quality is of utmost important at checkstyle and several checks are run on a PR before it could be merged which also helps improve the developer's skills
  • Divide and Conquer. During the initial phases it so used to happen that the produced work resulted in a lot of diff counts during regression testing making the corresponding diff reports huge and making it easy to miss something here or there. Later on, as I learnt splitting work into easily achievable tasks it became evident that having several small tasks speeds up the developement much more than working in bigger chunks. This also eases things for the reviewers resulting in quicker acceptance of code.
  • Several objectives had to be achieved in the short span of the project period. Chasing deadlines imposed a greated need for parallelism in work.
  • Insight into parsing theory and working with ANTLR.
  • Walking through specifications. The work mainly related to javadoc parsing in checkstyle. Grammar , that's used for parsing design is to be based on corresponding specifications. The grammar mainly appeals to the HTML specification and the oracle documentation guidelines. Discussions for designing anything tend to take a lot of time which sort of demands one's efficient at quickly extracting the details from specifications and presenting the alternative designs that can be concluded.
